## 2. 已验证的结论

每条都附了证据和复现方法。

### 2.1 官方 ATTACH_ANGLE 的定义

`hardware/makeyourpet-hexapod/chica-config-2040.txt:87-90`：

```
# The angle between the servo itself and the leg segment when the servo is centered.
COXA_ATTACH_ANGLE -8
FEMUR_ATTACH_ANGLE 35
TIBIA_ATTACH_ANGLE 68
```

### 2.2 -8/35/68 物理上装不准，这不是问题

DS3235 是 25 齿花键 → 360/25 = **14.4°/格**，最坏情况残差 **±7.2°**。
官方这三个数是设计意图，不是公差要求。

`docs/P0-GUIDE.md` 常见问题表里其实已经写了正确做法：
> 「舵盘花键有齿距，装最接近的一格，残差以后用 `config.py` 的 `attach_deg` 修」

**结论：舵盘不需要拆重装。** 需要做的是实测 `attach_deg` 填进 config。

### 2.3 行程余量充足（2026-07-17 已按实测值重算）

按实测 config（`tibia_len=120`、tibia attach k=93.6、femur attach α=49.7）算地面行走
（`max_step=60`、`step_height=40`、站高 90）的关节角包络：

| 关节 | 相对中位需要的行程 | 离 ±90° 极限的余量 |
|---|---|---|
| femur | -33.0° ~ +9.7° | 57.0° / 80.3° |
| tibia（k） | -6.6° ~ +48.0° | 83.4° / 42.0° |

最紧处仍有 42° 余量。**结论维持：舵盘不用拆。**

### 2.4 ★ tibia 的膝轴位置（本轮最硬的一条）

`left-tibia.stl` 上 4 个 M1.6 舵盘螺丝孔（Ø1.69）共圆：

```
圆心 (47.77, -14.59)   节圆半径 24.52mm   拟合残差 0.002mm
```

用 y=-4 和 y=-12 两个独立切片互验，圆心差 0.07mm。

**→ 膝轴 K = (47.8, -14.6)，在 tibia 坐标系里，不是原点。**

那个看起来像"7"字钩的上半段，**就是舵盘安装盘**（Ø49 节圆正好铺满它）。

复现：
```bash
.venv/bin/python - <<'EOF'
import numpy as np, trimesh, math
m = trimesh.load("hardware/makeyourpet-hexapod/STL/left-tibia.stl")
sec = m.section(plane_origin=[0,-4,0], plane_normal=[0,1,0])
pts=[]
for e in sec.entities:
    v = sec.vertices[e.points][:, [0,2]]          # 注意：必须直接取 3D 的 x,z
    c = v.mean(0); r = np.hypot(*(v-c).T)         # to_2D() 每片坐标系随机，不能用
    if r.std() < 0.3 and 1.0 < r.mean()*2 < 8: pts.append(c)
P = np.array(pts)
A = np.c_[2*P[:,0], 2*P[:,1], np.ones(len(P))]
sol, *_ = np.linalg.lstsq(A, (P**2).sum(1), rcond=None)
print("膝轴 =", sol[:2], " R =", math.sqrt(sol[2]+sol[0]**2+sol[1]**2))
EOF
```

### 2.5 femur 的连杆线方向

- `left-femur.stl` 叉口 42mm ≈ DS3235 本体 40mm → **叉口方向 = 膝轴方向**。
- 连杆线沿 STL 的 X 轴（两轴 x=0 与 x=80，对应 `FEMUR_LEN 80`）。
- 与连杆线**精确平行（0.00°）**的长直边存在（XY 投影：y=+9 长 80.4mm，y=-7 长 82.2mm）。

⚠️ femur 轴在 STL 里的**具体坐标未做 §2.4 那样的孔位验证**，只是从 x 跨度和 FEMUR_LEN=80 推的。
方向可信，**位置存疑**——需要用同样的孔位圆拟合验一遍。

### 2.9 robot.py 与 config.py 的 tibia 基准可能差 44°

`software/hexapod/robot.py:63-64`：
```python
# 膝关节变量取弯曲角 k = 180 - theta（伸直=0），与 TIBIA_ATTACH_ANGLE 同基准
arr[leg.tibia.channel] = leg.tibia.joint_deg_to_us(180.0 - math.degrees(th))
```
配 `config.py` 的 `attach_deg=68` → **软件认定舵机回中时膝内角 theta = 112°**（钝角，腿near伸直）。

FK 反证（用 `alpha=35°` 与两种基准分别代入）：

| 解释 | 回中时足端 | 合理性 |
|---|---|---|
| 68 = theta | 髋外伸 138.7mm，髋高 84.7mm | 正常站姿 |
| 68 = k=180-theta（现代码） | 髋外伸 220.9mm，髋高 **27.1mm** | 肚皮贴地，不像站姿 |

且 femur 侧支持「回中≈站立」：站立算得 `alpha=32.2°` vs 官方 35°，差 2.8°。

**倾向结论：68 是 theta 基准，现在的代码恒偏 44°。** 但此检验用了 `tibia_len=134`
（官方原版腿的值，用于判断官方意图是合理的），仍建议用实测 theta 复核后再改。

若成立，修法二选一：
- `config.py` tibia `attach_deg`: 68 → 112（保持 robot.py 传 k）
- 或 robot.py 改传 theta 本身，attach 保持 68、`sign` 取 -1

### 2.10 吸盘会被大气压压扁

-50kPa × Ø30 = **35.3N**，远大于手按的预压力。
→ 吸住后波纹继续压缩，**真空度越高腿越短**。
→ 严格说「膝轴到墙面的距离」是压力的函数，不是几何常数。

### 2.11 L1 三边实测与 theta_center（2026-07-17，整 mm 粗测）

前提：tibia 通道 17 @ 1510µs、全程使能；P 取**吸盘唇口圆心（自由态）**。
方法：量 F（femur 舵盘中心螺丝）、K（tibia 舵盘中心螺丝）、P 三点两两距离，余弦定理定角，
不依赖任何实体边（规避 §4.2 的"没有平行面"问题）。

```
FK = 80    与 FEMUR_LEN=80 吻合 → 取点方法自检通过
KP = 120   该口径下的 tibia_len 候选值
FP = 140
theta_center = arccos((80²+120²−140²)/(2·80·120)) = arccos(0.0625) ≈ 86.4°
```

- **精度**：整 mm 读数；边长各差 1mm 时 θ 约差 1.2°。填 config 前建议卡尺复测到 0.5mm 内。
- **换算**：保持 robot.py 传 k → L1 tibia `attach_deg = 180 − 86.4 = 93.6`；
  改 robot.py 传 θ 本身 → `attach_deg = 86.4`（配合 sign 调整）。二选一，勿混用。
- **§2.9 的官方基准问题本次判不动**：86.4° 离 68 差 18.4°（≈1.3 齿）、离 112 差 25.6°（≈1.8 齿），
  且装舵盘时本就没有严格对官方位。按"离最近齿"论弱支持 θ 基准（68+14.4=82.4，差 4°），不作结论。
  但对**已实测的腿**此问题不再有实际影响——实测值直接覆盖官方 68。
- **P 口径**：此测量把 §4.1 从"STL 推导题"变成了"口径选择题"——取唇口自由态圆心后，
  theta 与 tibia_len 同口径自洽，运动学闭环。STL 推导仍可做（§4.1），用于交叉验证。
- **交叉验证**：若 P 在方轴轴线上（x=1.3，§2.7 腔体与方轴同心），KP=120 ⇒ 唇口在 STL
  z ≈ −125.6，即折痕面（z=−100）下 25.6mm、打印件底（z=−110.95）下 14.6mm。
  这是 §4.4 自由态 h_cup 的第一个实测参考值（±2~3mm）。
- **2026-07-17 用户复核三个数可直接用，已应用**：`config.py` L1 tibia `attach_deg=93.6`
  （保持 robot.py 传 k）、`tibia_len=120`；`tools/workspace_analysis.py` 限位改为
  θ∈(0,176.4)；§2.3 行程包络与 CLIMBING-DESIGN §6 数表已重算，结论均不变；
  单元测试 18 passed。

### 2.12 L1 femur 高差法实测（2026-07-17）

前提：腿根（coxa 舵机壳底面）水平放平——P0 无机身，壳底平贴桌面即可让 coxa 轴竖直；
通道 16 @ 1510µs、全程使能；桌沿卡在 K 与吸盘之间
（F、K 留在桌面上方可量高，吸盘出檐悬空防堵转）。量桌面→F、桌面→K 螺丝顶
（同型号螺丝，头半径在差值中抵消）。

```
Δh = h_K − h_F = 61
α_center = arcsin(61/80) = 49.7°
```

- **49.7 ≈ 官方 35 + 一个花键齿 14.4 = 49.4（差 0.3°）**——正落在齿距网格上，
  与"装配时离官方位差一格"吻合，量法自洽。
- 精度：Δh ±0.5mm → 角度 ±0.55°（此处 cosα≈0.65，α 越大越敏感）。
- 已应用：config L1 femur `attach_deg=49.7`；`ALPHA_LIM=(-40.3,139.7)`；
  §2.3 包络与 CLIMBING-DESIGN §6 重算——上抬行程 125→139.7° 后，
  **平身地-墙过渡在墙距 D=100 从不可行变为可行**（+25~+85mm），其余结论不变。
- 旁证：femur 正落网格，反推 tibia 的 86.4°（离 θ 基准网格 82.4 差 4.0°）可能含 2~4°
  量测误差；但行走余量 ≥42°，无实际影响，暂不复测，P3 FK 抽查时一并验证。

## 4. 未决问题（按优先级）

### 4.1 ★ tibia 的足端 P 在哪？K→P 的方向是什么？

已知：K = (47.8, -14.6)；官方 `TIBIA_LEN 134`；方轴轴心 x=1.3；打印件底部 z=-110.95。

若假设 P 落在方轴轴线上（x=1.3）：
```
(47.77-1.3)² + (-14.59-z)² = 134²   →   z_foot = -140.3
```
即 P 在打印件底部**再往下 29.4mm**。`micro-switch-tip.stl` 高 29mm（Z: 7..36）——数字接近，
但**这仍然是旁证，不是验证**（§3.1 就是栽在这种推理上）。

此假设成立的话，K→P 与方轴夹角约 **20.3°**，即方轴/方盒子**不能**当基准面。

**必须先解决这条**，因为 §4.2、§4.3 都依赖它。

建议验证方向：
- 找 `left-tibia.stl` 下端方轴与 `micro-switch-tip.stl` / `tip.stl` 的配合特征，定出足端；
- 或直接在实物上量：K（舵盘中心螺丝）到足端的距离，反推。

### 4.2 tibia 上有没有平行于 K→P 的实体面？

§4.1 未定 → 此条未定。若没有，theta 的测量方式要重新设计（不能靠贴面）。

### 4.3 合体件的真实 tibia_len

`tibia_len=134` 确定是错的（§2.6），但**正确值未知**（依赖 §4.1）。
且需先决定 P 的建模口径：

- **P = 刚性点**（零件上某个固定位置）→ `l3` 是常数，吸盘的压缩量 `h_cup` 单独建模
- **P = 接触平面** → `l3 = 常数 + h_cup(真空度)`，不是常数

后者会把一个变量伪装成常数：地面走（0kPa）和墙上吸住（-50kPa）的 `l3` 差约 10mm，
一个常数不可能两边都对。倾向前者，但需在 §4.1 解决后重新论证。

### 4.4 h_cup（折痕面 → 吸盘唇口）随真空度的变化

需要 P1 的真空回路才能测。至少三个态：
| 状态 | 用途 |
|---|---|
| 自由（无外力） | 摆动相：判断何时接触墙面 |
| 压到密封、未抽气 | 预压行程 |
| 抽到 -40~-50kPa 稳定 | 支撑相 IK（承力工况） |

前两者之差 = 预压行程，是 P2「伸腿→压紧→抽气」的设计参数。

### 4.5 femur 轴在 STL 里的精确位置

§2.5 只验了方向，没验位置。用 §2.4 的孔位圆拟合方法验一遍。

### 4.6 theta / attach_deg 的实测

依赖 §4.1、§4.2（决定用什么基准面量）。

已知的量测约束：
- 必须发 **1510µs** 而不是 1500µs。`attach_deg` 的定义点是标定中心
  `(us_m45+us_p45)/2 = (1040+1980)/2 = 1510`，而 `scripts/servo_center.py:25` 发的是 1500，差约 1°。
  用 `scripts/servo_calib_helper.py` 敲 `17 1510`。
- 全程舵机保持使能，不能用手掰（35kg 堵转会打齿轮，且掰完角度是假的）。
- 精度分档：定 §2.9 那个 44° 基准矛盾只需 ±20°；填 config 需 ±1~2°
  （P3 验收 IK <5mm，在 ~130mm 力臂上约 2.2°）。

### 4.7 修正后的连带影响

`tools/workspace_analysis.py:23-24` 的 `ALPHA_LIM=(-55,125)` / `THETA_LIM=(22,180)`
是从 attach 35/68 硬编码推出来的（35±90 等），且 `THETA_LIM` 用的是 k 基准（与 §2.9 相关）。
`attach_deg` 或 `tibia_len` 一改，这两个限位要跟着改，
`docs/CLIMBING-DESIGN.md` 的「可落足带」结论（P5 地-墙过渡的依据）要重算。
