# 爬墙蜘蛛机器人 · 实现路线图

> 总体策略：**最大风险前置——先用最少的钱验证"单足上墙"，通过决策门后才投入整机。**
> 地面行走是 MakeYourPet 社区验证过的成熟路线，信息量低，放到吸附验证之后做。
> 采购分两批：第一批只买单腿验证套件（约 ¥600~950），15 个舵机等大头在 P2 决策门通过后才下单。
> 爬墙对重量极其敏感，每个阶段都要记录重量预算。

## 技术路线

**吸盘足 + 电磁阀组**：每只脚一个真空吸盘，泵抽真空吸附，电磁阀放气释放。
能静态停留（常闭阀断电保持真空、泵间歇工作）、可负重、"蜘蛛侠"式真爬墙；
整机重量预算 ~2.5kg（3 只 30mm 吸盘同时吸附即 >100N 法向力）。
限制：只适合光滑墙面（玻璃、瓷砖、烤漆金属、亚克力）。
代价：步态必须与阀切换、压力反馈联动，控制是本项目的核心工作量（`software/` 已搭好骨架）。

> 曾评估过 EDF 涵道风扇压墙路线（适合粗糙墙面），已放弃：它只对 <800g 的平台成立，
> 与选定的 MakeYourPet 大平台（~2kg）物理上不兼容。论证见 `CLIMBING-DESIGN.md`。

## P0 准备（第 1~2 周）

> 逐步操作指南（含命令、核对表、常见问题）：[P0-GUIDE.md](P0-GUIDE.md)

- 下单**第一批**器件：单腿验证套件（见 `BOM.md` 开头"分批采购"），舵机只买 4 个（3 用 1 备）。
- 打印：一条完整腿——coxa/femur 用 `hardware/makeyourpet-hexapod/STL/`，tibia 用 `climbing-parts/left-tibia-suction.stl`（吸盘足一体化小腿）+ 1 件 `suction-foot-door.stl`。
- 树莓派 Pi 5 准备：烧录 Raspberry Pi OS Lite 64-bit、开 SSH/I2C，装 `software/` 包并跑通 `sim_walk.py` 仿真与 `pytest`。
- 验收标准：一条腿 3 个舵机装配完成，Pi 经 Servo2040 能摆动它；Pi 上仿真与测试全绿。

## P1 吸附系统台架验证（第 2~5 周）

> 逐步操作指南（含接线、代码、记录模板、常见问题）：[P1-GUIDE.md](P1-GUIDE.md)

先在工作台上把真空回路调通（水平桌面即可，还不上墙）：

- 搭建单足真空回路：泵 → 单向阀 → 储气罐（PET 瓶即可）→ 电磁阀 → 吸盘，接 XGZP6847A 压力传感器；Pi 5 经 ADS1115 读压力、GPIO 驱动 MOSFET 开关泵/阀（实现 `software/hexapod/adhesion.py` 里的 `Pi5VacuumIO`，状态机和仿真已就绪）。
- 在竖直玻璃/瓷砖上测试，记录三条曲线：
  1. 抽气时间：吸盘贴壁 → -40kPa 需要多久（目标 <1s）
  2. 泄漏率：阀关闭后压力回升速度（目标 -40kPa 保持 >60s 不掉到 -20kPa）
  3. 释放时间：阀放气 → 吸盘可无阻力拿开（目标 <0.5s）
- 破坏性测试：单只 30mm 吸盘垂直挂重、剪切挂重各多少 kg 才脱落（理论法向 ~35N@-50kPa，剪切约打 5 折）。
- **斜角吸附测试**（活动度分析的关键输入，见 `CLIMBING-DESIGN.md` §6）：吸盘轴相对墙面法线倾 5°/10°/15°/20° 时的密封成功率与拉脱力。实测容差 ≥15° 则爬墙步幅可用 40mm；只有 10° 则收紧到 ~25mm。
- 吸盘装进 `left-tibia-suction.stl` 腔体并粘上门盖后重复上述测试（验证打印件夹持刚性和强度）。
- **验收标准：单吸盘足在竖直玻璃上挂 3kg 配重保持 10 分钟；一个"吸-确认-放"循环 <2s。**

## P2 单腿爬墙试验（第 5~9 周）——决策门

> 逐步操作指南（含台架图、几何常量、分级测试、常见问题）：[P2-GUIDE.md](P2-GUIDE.md)

整个项目最关键的验证，通过之前不买第二批器件：

- **试验台**：木板/铝型材搭 L 形支架，把髋关节（coxa 轴）固定在竖直玻璃前约 130mm 处（腿工作空间中位）；腿 + 吸盘足对墙做"伸腿 → 压紧 → 抽气 → 压力确认 → 承力 → 放气 → 抬腿"完整循环。
- **承力模拟**：把腿的安装座改装在一个可沿竖直导轨滑动的滑车上，滑车配重 1.5kg（≈ 整机 2.5kg ÷ 3 只有效承力足，加余量）。吸附确认后松开滑车锁——这条腿必须独立撑住配重，这就是它在整机上墙时的真实工况。
- 软件交付物：`scripts/single_leg_wall.py`（单腿 IK + `adhesion.py` 状态机联调：只有压力 ≤-30kPa 才转移配重；失败自动抬 5mm 重试），记录每次循环的压力曲线和舵机电流。
- 测出舵机在墙面剪切载荷姿态下的真实力矩余量（35kg·cm 舵机按 50% 降额使用）。
- **验收标准（决策门）：连续 50 次吸放循环成功率 >95%；吸附状态下滑车 1.5kg 配重悬挂 5 分钟 + 施加 1kg 侧向拉力不脱落。**
- 决策：通过 → 下单第二批（15 个舵机 + 其余全部），进 P3；不通过 → 升级 40mm 吸盘/双泵重测；仍不通过 → 项目方案重估，此时只花了小几百块。

## P3 地面行走（第 9~15 周）

单腿验证通过后，把成熟路线走完（P1/P2 期间打印机闲着就可以先把整机结构件打出来）：

- 打印全部结构件（PLA 即可，腿部件建议 4 壁 + 40% 填充），装配 18 个舵机、走线、舵机中位校准。
- 控制软件上手顺序：
  1. `scripts/servo_center.py` 舵机回中、装舵盘、按 `software/README.md` 标定流程填 ±45° 脉宽；
  2. `scripts/stand_up.py` 站立；`scripts/walk_teleop.py` 键盘遥控三角步态。
- 足底限位开关接 Servo2040 传感器口（官方接线），Pi 经同一条 USB 读取，实现触地检测步态。
- **验收标准：平地直线行走 2m、原地转 360°、跨过 2cm 障碍、单腿 IK 点到点误差 <5mm。**
- 里程碑重量记录：称整机重量，超过 2.2kg 就要开始减重（爬墙预算上限 2.5kg 含吸附系统）。

## P4 整机爬墙集成（第 15~26 周）

- 6 只吸盘足上机；气路用 1 泵 + 储气罐 + 6 阀（阀组装在 `component_plate.stl` 上）；电磁阀选**常闭型**——断电时保持真空，机器人断电不掉。
- 步态从地面三角步态（3+3）改为保守的**五足支撑步态**（每次只挪 1 只脚），配合阀切换时序和压力确认。
- 结构姿态：爬墙时身体贴近墙面（重心距墙越近，上排吸盘的剥离力矩越小）。
- **全程系安全绳**（顶部打点，登山辅绳 + 轻微余量），地面铺垫子。
- 地-墙过渡（从地面走到墙上）是最难的动作，此阶段跳过：人工把机器人放上墙。
- **验收标准：竖直玻璃/瓷砖墙上，向上爬 1m、水平横移 0.5m、原地断电悬停 5 分钟不脱落。**

## P5 负重与能力扩展（第 26 周以后，按需排序）

1. **负重**：目标挂载自重 20~30%（400~600g）。手段：吸盘升到 40mm（法向力 ×1.8）、泵升级双头、砝码递增测试找出安全边界的 50% 作为额定负载。
2. **地-墙过渡**：前两足先上墙吸附 → 身体后仰 → 中足上墙 → 后足上墙。活动度分析（`CLIMBING-DESIGN.md` §6）已确认运动学上有解：平身时墙距前髋 140~200mm 存在约 70mm 高的可落足带，身体俯仰 30° 后更宽；难点在过渡序列的编排与重心管理，不需要改腿几何。
3. **感知与遥控升级**：Pi 5 CSI 口接摄像头模块做 FPV；MPU6050 姿态检测（检测到滑坠立即全阀吸附 + 报警）。
4. **天花板行走**：吸盘方案理论可行，法向力预算需重算（全部重量变成剥离力）。
5. **自主化**：SLAM/循线在墙面场景意义不大，优先做"一键回充/一键下墙"这类实用自动动作。

## 风险清单（按杀伤力排序）

| 风险 | 缓解 |
|---|---|
| 投入全部预算后才发现吸附方案不可行 | 阶段重排已规避：P2 单腿决策门通过前只花第一批的钱（约 ¥600~950） |
| 整机太重，吸附余量不足 | 每阶段称重；P3 结束超 2.2kg 即减重（去外壳、换 2S 小电池） |
| 阀/管路泄漏，吸附时间不够走一步 | P1 台架把泄漏率测清楚再上机；接头全部用卡扣+扎带+密封胶 |
| 舵机在墙面姿态下力矩不够/过热 | P2 单腿实测；35kg·cm 舵机 50% 降额；步态放慢 |
| 墙面灰尘导致吸盘失效 | 只在擦净的玻璃/瓷砖上测试；吸盘唇口定期酒精清洁 |
| Pi 5 供电不稳，行走/爬墙中重启 | 独立 5V/5A 降压模块供电，绝不与舵机共轨；`vcgencmd get_throttled` 验证；欠压历史非零就先解决供电再继续 |
| 摔机损失 | 安全绳全程不摘；先 0.3m 低高度测试 |
| 一开始就做地-墙过渡 | 不要。P4 人工上墙，过渡放 P5 |
